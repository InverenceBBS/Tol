# README - doctools Multi-Part Formatters (tcllib 1.10)

## Overview

This directory contains the multi-part formatters (plugins) for the `doctools` documentation generation system within Tcllib 1.10, located in the `tolsh` VFS. These formatters are responsible for converting the intermediate representation generated by `doctools` into final, human-readable documentation formats.

## Contents

### Common Files

- `_common.tcl`: Common procedures shared by multiple formatters.
- `_html.tcl`: Common procedures specific to HTML-based formatters.
- `_nroff.tcl`: Common procedures specific to nroff-based formatters.
- `_text.tcl`: Common procedures specific to plain text formatters.
- `_xml.tcl`: Common procedures specific to XML-based formatters.
- `_idx_common.tcl`: Common procedures for index generation across formats.
- `_toc_common.tcl`: Common procedures for table-of-contents generation across formats.

### Message Files

- `*.msg` (e.g., `en.msg`, `de.msg`, `fr.msg`): Language-specific message catalogs used by the formatters for localization.

### Main Formatters (`fmt.*`)

These files implement the logic for specific output formats:
- `fmt.html`: Generates HTML output.
- `fmt.latex`: Generates LaTeX output.
- `fmt.list`: Generates a simple list format.
- `fmt.nroff`: Generates nroff (troff) output, suitable for man pages.
- `fmt.null`: A null formatter (produces no output).
- `fmt.text`: Generates plain text output.
- `fmt.tmml`: Generates TMML (Tcl Manpage Markup Language) output.
- `fmt.wiki`: Generates output suitable for wiki markup.
- `fmt.desc`: Possibly related to format descriptions or metadata.

### Index Formatters (`idx.*`)

These files handle the generation of index files in specific formats:
- `idx.html`, `idx.nroff`, `idx.null`, `idx.text`, `idx.wiki`

### Table of Contents Formatters (`toc.*`)

These files handle the generation of tables of contents in specific formats:
- `toc.html`, `toc.nroff`, `toc.null`, `toc.text`, `toc.tmml`, `toc.wiki`

### Subdirectories

None.

## Purpose

The purpose of this directory is to provide the output generation capabilities for the `doctools` system. By separating the formatting logic into these plugins, `doctools` can easily support multiple output formats and potentially be extended with new ones. Each formatter takes the parsed documentation structure and renders it according to the rules of the target format.
